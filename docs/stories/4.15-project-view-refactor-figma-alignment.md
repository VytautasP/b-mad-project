# Story 4.15: Project View Refactor — Figma Alignment

## Status

Ready for Review

## Story

**As a** project-focused user,
**I want** the Project view to present tasks in a structured table layout matching the approved Figma design,
**so that** I can scan, organise, and manage project tasks efficiently in a polished, design-consistent interface that replaces the placeholder page.

## Current State Analysis

| Aspect | Current | Target |
|--------|---------|--------|
| **ProjectComponent** | Placeholder — inline template: `<h1>Project</h1><p>Tree view coming soon.</p>` | Full table-based project view per Figma |
| **TaskTreeComponent** | Nested `mat-tree` with drag-drop in Dashboard view-mode toggle | Not used in new Project view; Dashboard keeps its own tree toggle |
| **Route** | `/project` → `ProjectComponent` (lazy, auth-guarded) | Same route, fully implemented component |
| **Data source** | None (placeholder) | `TaskService.getTasks()` — same service the tree and list views use |
| **Hierarchy display** | Nested tree nodes via `NestedTreeControl` + `MatTreeNestedDataSource` | Flat table rows with indent + breadcrumb path to convey hierarchy |

### Key Architectural Decisions

1. **New component, new template** — `ProjectComponent` gets its own `.html` and `.scss` files (replace inline template).
2. **Table-based layout** — The Figma design shows a flat table (not a `mat-tree`), with hierarchy communicated via indentation and breadcrumb text in the Task Name column.
3. **Reuse `TaskService`** — Fetch tasks the same way the tree and list views do; convert the flat task array into display rows with hierarchy metadata (parent path / indent level).
4. **Do not break Dashboard** — `TaskTreeComponent` continues to be imported and used in `DashboardComponent`'s tree view mode. This story only builds out the Project route.
5. **Shared design tokens** — Reuse status badges, priority colours, card styling, and typography tokens established in Story 4.10.

## Acceptance Criteria

1. **Page heading** renders a project title area with search input and primary action button matching the Figma heading section (h = 109, white bg, bottom border).
2. **Sub-header / filter bar** renders below the heading (h = 59, white bg, bottom border) with left and right content containers for filters / controls.
3. **Table card wrapper** uses `border-radius: 12px`, `border: 1px solid #E2E8F0`, and `box-shadow: 0 1px 2px rgba(0,0,0,0.05)`.
4. **Sticky table header** is fixed at the top of the card on scroll with `backdrop-filter: blur(4px)`, semi-transparent background, and uppercase bold column labels (TASK NAME, STATUS, DUE DATE).
5. **Table column order** matches Figma: Checkbox → Task Name (with hierarchy indent + breadcrumb) → Status → Due Date → Avatars → Actions.
6. **Checkbox column** is present as the first column with a `24×24` checkbox (`border-radius: 4px`) enabling row selection.
7. **Task Name cells** display a task-type icon, task title, and a breadcrumb/hierarchy path conveying the parent chain — with `padding-left` to indicate nesting depth.
8. **Status cells** render as colour-coded pill badges with dot indicator, reusing the design tokens from Story 4.10 (In Progress = blue, To Do = gray, Done = green).
9. **Due Date cells** display dates in `MMM dd, yyyy` format (Manrope 400 / 14px / `#64748B`).
10. **Avatars column** shows stacked circular avatar indicators for assignees.
11. **Actions column** contains a three-dot overflow menu, hidden by default (`opacity: 0`), visible on row hover.
12. **Expandable rows** allow grouped or child tasks to expand/collapse inline within the table.
13. **Pagination footer** renders inside the card with "Showing X–Y of Z tasks" text and Previous / Next buttons styled per Figma.
14. **Data integration** uses `TaskService` to fetch tasks and builds a flat row model with hierarchy metadata (indent level, parent breadcrumb path) from the existing task `parentId` relationship.
15. **Existing routes and components are unaffected** — Dashboard tree/list toggle, task detail navigation, sidebar active states all continue to work.
16. **Typography** uses Manrope at specified weights/sizes; column headers use Bold 700 / 12px / uppercase with `0.6px` letter-spacing.
17. **Sidebar active state** highlights "Project" nav item with the blue 10 % opacity background when on the `/project` route (verify `routerLinkActive` works correctly).
