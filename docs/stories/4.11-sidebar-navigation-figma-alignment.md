# Story 4.11: Sidebar Navigation — Figma Alignment

## Status

Ready for Review

## Story

**As a** TaskFlow user,
**I want** a persistent left-side navigation sidebar that matches the approved Figma design,
**so that** I can quickly switch between Tasks, Project, Timeline, Logs, and Settings views from any authenticated page with clear visual feedback on which view is active.

## Design Reference

- **Figma file:** [TaskFlow — Sidebar Navigation] file on figma_local mcp, node-id=7:42
- **Node:** `7:42` (`Aside` frame)
- **Appears on:** Every authenticated page — My Tasks ListView (`7:2`), Task Full Details (`7:280`), Tree View (`7:633`), Timeline (`7:1048`), Timeline Empty State (`7:1395`), Time Tracking Logs (`7:1526`)

## Figma Design Summary

The Figma design specifies a fixed-width left sidebar that replaces the current top `mat-toolbar` navigation for authenticated views. The sidebar persists across all app pages (except login/register) and contains primary navigation links plus a bottom-pinned Settings link.

### Sidebar Container (Aside)
- **Width:** 256px fixed
- **Height:** Full viewport height (below header, or full height if no top header — per page Figma frames)
- **Background:** White (`#FFFFFF`)
- **Right border:** 1px solid `#E2E8F0` (Slate-200)

### Inner Container
- **Padding:** 24px top / 24px bottom
- **Layout:** Vertical, fill height

### Nav Section
- **Padding:** 16px left / 16px right
- **Item spacing:** 4px vertical gap between links
- **Contains 4 primary links:**
  1. **Tasks** (icon + label) — maps to `/dashboard` (My Tasks list view)
  2. **Project** (icon + label) — maps to future `/project` route (tree view)
  3. **Timeline** (icon + label) — maps to `/timeline`
  4. **Logs** (icon + label) — maps to future `/logs` route (time tracking logs)

### Nav Link — Active State
- **Background:** `rgba(19, 127, 236, 0.1)` (Brand blue at 10% opacity)
- **Text color:** `#137FEC` (Brand blue)
- **Icon color:** `#137FEC` (Brand blue)
- **Border radius:** 8px
- **Padding:** 12px horizontal, 10px vertical
- **Gap (icon to text):** 12px
- **Full width** (stretches to fill nav container)

### Nav Link — Default State
- **Background:** Transparent
- **Text color:** `#64748B` (Slate-500)
- **Icon color:** `#64748B` (Slate-500)
- **Border radius:** 8px
- **Padding:** 12px horizontal, 10px vertical
- **Gap (icon to text):** 12px

### Typography (All Links)
- **Font:** Manrope Medium (500)
- **Size:** 14px
- **Line-height:** 20px

### Bottom Section (Settings)
- **Separated** from main nav by a top border: 1px solid `#E2E8F0`
- **Padding:** 24px top, 16px left/right (within border container)
- **Contains:** "Settings" link with icon, same link style as default nav links
- **Position:** Pushed to bottom via spacer/flex-grow on nav section

### Icon Sizing
- Nav icons: ~16.67 × 16.67px (except Logs at 15 × 15px)
- Settings icon: ~16.75 × 16.67px

## Acceptance Criteria

1. **Sidebar component** is created as a standalone Angular component at `src/app/shared/components/sidebar/` replacing the current top `mat-toolbar` navigation for authenticated views.
2. **Sidebar layout** renders as a 256px fixed-width panel on the left, with the main content area filling the remaining viewport width to the right.
3. **App shell layout** changes from vertical (top-nav → content) to horizontal (sidebar | content) for all authenticated routes; login and register pages continue to render without the sidebar.
4. **Five navigation links** are rendered: Tasks, Project, Timeline, Logs (main nav section), and Settings (bottom section separated by a border).
5. **Active route highlighting** applies the blue background/text style to the link matching the current route; all other links use the default slate style.
6. **Active state uses `routerLinkActive`** directive for automatic route-based highlighting.
7. **Design tokens match Figma exactly:** white background, `#E2E8F0` right border, 8px link border-radius, `#137FEC` active color at 10% opacity bg, `#64748B` default text, Manrope Medium 14px/20px, 12px icon-to-text gap, 4px inter-link gap, 16px horizontal nav padding, 24px vertical container padding.
8. **Settings link** is visually separated at the bottom by a 1px `#E2E8F0` top border with 24px top padding.
9. **Icons** are used from Material Icons at approximately 16–17px size, matching the visual intent of each Figma icon (checkmark/list for Tasks, folder for Project, calendar/clock for Timeline, document/list for Logs, gear/cog for Settings).
10. **Current user info and logout** functionality (from the existing `NavigationComponent`) is preserved — either integrated into the sidebar (e.g., user avatar/name and logout in the bottom section) or placed in a minimal top bar.
11. **Existing routing and auth guard behavior** is preserved — all protected routes remain guarded, login/register remain public, default redirect to `/dashboard` is unchanged.
12. **Responsive behavior:** On mobile/tablet breakpoints (below 1024px), the sidebar collapses or hides behind a hamburger toggle to avoid consuming excessive screen space. The collapsed state should still provide access to navigation.
13. **Accessibility:** All nav links are keyboard-navigable, have visible focus indicators, use `<nav>` landmark with `aria-label`, and active link state is communicated to screen readers (e.g., `aria-current="page"`).
14. **No regression** in existing page functionality — task list, task details, timeline, create task, auth flows all continue to work.

## Dev Notes

### Current State
- The app currently uses a top `mat-toolbar` navigation (`src/app/shared/components/navigation/`) with "TaskFlow" brand, user name, and logout button.
- The app shell in `app.html` renders: `@if (showNavigation()) { <app-navigation> }` + `<router-outlet>` + `<app-timer-widget>` in a vertical layout.
- The `showNavigation` signal hides nav on login/register routes — this logic should be reused for the sidebar.

### Figma Context
- The sidebar (Aside) is present on **all authenticated page designs** in Figma (nodes `7:42`, `7:1396`, `7:1527`, etc.) confirming it is a global app shell component, not page-specific.
- The Figma pages use the layout: `Body → [Header (top bar)] + [Container → [Aside (sidebar) | Main (content)]]` or simply `Body → [Aside | Container]` depending on the page.
- Some Figma pages show a top Header bar alongside the sidebar; the implementation can include a minimal top header or integrate all header concerns into the sidebar.
