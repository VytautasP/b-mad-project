# Story 4.13: Quick Inspect Modal — Refactor & Figma Alignment

## Status

Approved

## Story

**As a** TaskFlow user,
**I want** a polished, visually consistent Quick Inspect modal when I click on a task in the list view,
**so that** I can quickly review a task's key details (title, status, description, priority, due date, assignees) and decide whether to open it fully, edit it, or archive it — all in a design that matches the approved Figma specification.

## Design Reference

- **Figma file:** TaskFlow — Task Quick Inspect Modal via figma_local MCP server
- **Node:** `1:413` (`Task - Quick inspect modal` page, containing `1:423` Modal Container at 512 × 475px)
- **Triggered from:** Task list row click, tree view task click, timeline task click

## Figma Design Summary

The Figma design specifies a centered modal dialog that overlays the current view with a semi-transparent dark backdrop with blur. The modal has three distinct sections: a **header** (title + status), a **body** (description + metadata + assignees), and a **footer** (archive + action buttons).

## Acceptance Criteria

1. **Modal overlay** renders as a full-viewport dark backdrop (`rgba(15, 23, 42, 0.6)`) with `backdrop-filter: blur(4px)` when the Quick Inspect modal is opened.
2. **Modal container** is centered, 512px max-width, white background, 12px border-radius, 1px `#F1F5F9` border, and `box-shadow: 0 25px 50px -12px rgba(0, 0, 0, 0.25)`. Content clips to border-radius.
3. **Header section** displays the task title in Manrope Bold 24px with -0.6px letter-spacing and `#0F172A` color, alongside a 30px circular close button with an X icon.
4. **Status badge** renders as a colored pill below the title: pill shape (border-radius 9999), icon + status text, with color-coded backgrounds per status (In Progress = blue `#EFF6FF`/`#137FEC`, Done = green, Blocked = red, etc.).
5. **Task ID** is shown next to the status badge with the task's identifier string (e.g., "TF-421").
6. **Description** displays the task description text in Manrope Regular 14px, `#475569` color, truncated to ~4 lines with ellipsis if longer.
7. **Metadata grid** shows Priority and Due Date in a two-column horizontal layout below the description, separated by a 1px `#F1F5F9` top border. Labels are uppercase Manrope Bold 12px `#64748B` with 0.6px letter-spacing. Values show icon + text in 14px.
8. **Assignees section** shows label + overlapping avatar circles (36px, -8px overlap, 2px white border) with a "+N more" overflow indicator when more than 4 assignees exist.
9. **Footer** renders with `#F8FAFC` background, 1px `#F1F5F9` top border, containing: (left) an "Archive" text button with icon in `#64748B`, and (right) an "Edit Details" ghost button + "Open Full Details" primary button in `#137FEC` with trailing arrow icon.
10. **"Open Full Details" button** navigates to the full task details page (`/tasks/:id`) and closes the modal — preserving existing behavior.
11. **"Edit Details" button** navigates to the task edit view or opens the task form dialog for quick editing, then closes the modal.
12. **"Archive" button** triggers a confirmation dialog and calls the archive/soft-delete endpoint for the task, closing the modal on success.
13. **Close button (X)** and pressing `Escape` close the modal. Focus returns to the element that triggered the modal.
14. **Task data** is dynamically sourced from the `Task` model passed via `MAT_DIALOG_DATA` — title, status, priority, due date, description, and assignees are all rendered from the task object.
15. **Design tokens match Figma exactly:** per the Figma Design Summary and Design Tokens table below — all colors, spacing, typography, border-radius, and shadow values.
16. **Accessibility:** The modal uses `role="dialog"` with `aria-modal="true"` and `aria-labelledby` referencing the task title. All buttons have `aria-label` attributes. Focus is trapped within the modal. The close button is keyboard-accessible.
17. **Responsive behavior:** On viewports below 640px, the modal stretches to near full-width with 16px side margins. The metadata grid stacks vertically when the modal is narrow.
18. **No regression** in existing functionality — opening from list view, tree view, timeline, task details page navigation, and escape/close behavior all continue to work correctly.

## Dev Notes

### Current State
- The Quick Inspect modal was initially split from the full details page in Story 4.1.
- The current component is at `src/app/features/tasks/task-detail-dialog/` — uses `TaskDetailDialog`, a Material Dialog with `mat-dialog-title`, `mat-dialog-content`, and `mat-dialog-actions`.
- Current layout: icon + title (h2) header → 2×2 property grid (Status, Priority, Due Date, Assignees) → optional description block → single "Open full details" button.
- Current styling uses Material default dialog chrome; colors are generic Material palette, not Figma-aligned.
- The dialog is opened from the task list via `MatDialog.open(TaskDetailDialog, { data: { task, openerElement } })`.

### What Changes
- **Header:** Replace flat icon+title row with a rich title (larger, bold) + separate status/task-ID badge row.
- **Body:** Replace 2×2 grid with description-first layout, then a two-column metadata grid (Priority + Due Date only), then a dedicated assignees section with overlapping avatars.
- **Footer:** Replace single button with three actions: Archive (left), Edit Details (ghost) + Open Full Details (primary, with arrow icon) on right.
- **Overlay:** Custom dark backdrop with blur instead of default Material backdrop.
- **Styling:** Full Figma token alignment — Manrope typography, exact spacing, exact colors.

### Figma Context
- The Quick Inspect Modal (node `1:413`) is designed to appear when a user clicks a task row in the list view, providing a fast summary without navigating away.
- It complements the full Task Details Page (route-based) and the Task List View (Story 4.10).
- The modal shares design language with the Shellbar (Story 4.12) and Sidebar (Story 4.11) — same Manrope font, Slate color palette, Brand blue `#137FEC`.
