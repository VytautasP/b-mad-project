# Story 4.12: Shellbar (Header) — Reusable Component Refactor & Figma Alignment

## Status

Ready for Review

## Story

**As a** TaskFlow user,
**I want** a consistent, polished top header bar (shellbar) across all authenticated pages that matches the approved Figma design,
**so that** I always have access to global search, a quick-create action, notifications, and my profile/account controls regardless of which view I'm on.

## Design Reference

- **Figma file:** TaskFlow — Header (Shellbar) via figma_local MCP server
- **Node:** `7:3` (`Header` frame, 1280 × 64px)
- **Appears on:** Every authenticated page alongside the sidebar — My Tasks ListView, Task Full Details, Tree View, Timeline, Time Tracking Logs, etc.

## Figma Design Summary

The Figma design specifies a full-width top header bar (shellbar) that sits above the main content area and spans the full viewport width. It is a horizontal layout with two main zones: a **left zone** (branding + search) and a **right zone** (actions + profile).

## Acceptance Criteria

1. **Reusable shellbar component** is created at `src/app/shared/components/shellbar/` as a standalone Angular component that can be included in the app shell layout for all authenticated pages.
2. **Shellbar layout** renders as a 64px fixed-height bar at the top of the viewport, spanning the full width (to the right of the sidebar if sidebar is present). It uses `space-between` horizontal layout with 24px horizontal padding.
3. **Brand section** displays the TaskFlow logo (32px blue rounded square with white icon) and "TaskFlow" text in Manrope Bold 18px, with 12px gap between them.
4. **Search input** renders as a 384px-wide, 38px-tall rounded input with light gray background (`#F6F7F8`), slate border, search icon overlay (left), and placeholder "Search tasks, projects, people..." in Manrope Regular 14px. The search input emits a value for future wiring to a search feature.
5. **"Create Task" button** is styled as the primary CTA with brand blue background, white text/icon, 8px border-radius, subtle drop shadow, and triggers navigation to the create-task flow or opens a create-task dialog.
6. **Vertical divider** separates the CTA from the notification/profile section — 1px wide, 24px tall, `#E2E8F0` color.
7. **Notification bell** icon button renders with a bell icon in slate color, with 8px padding and 8px border-radius, clickable placeholder for future notification panel.
8. **User profile section** displays the current user's avatar (32px circle), name (Manrope Medium 14px), and role/title (Manrope Regular 12px) with a dropdown caret icon. Clicking the profile section opens a dropdown menu with at minimum a "Logout" option.
9. **User data** is dynamically sourced from `AuthService.currentUser$` — avatar, display name, and role/title are rendered from the authenticated user model.
10. **Logout functionality** from the existing `NavigationComponent` is fully preserved and accessible from the user profile dropdown in the shellbar.
11. **App shell integration** updates `app.html` to render the shellbar above the sidebar+content area for authenticated views. The layout becomes: `shellbar (top, full-width)` → `sidebar (left) | content (right)`.
12. **The old `NavigationComponent`** (`src/app/shared/components/navigation/`) is removed or deprecated, with all its responsibilities migrated to the shellbar and sidebar.
13. **Responsive behavior:** On viewports below 1024px, the search input collapses (hidden or icon-only), the "Create Task" text may collapse to icon-only, and the user profile section shows only the avatar. The shellbar height remains 64px.
14. **Accessibility:** The shellbar uses a `<header>` landmark with appropriate `role` and `aria-label`. The search input has an associated label. All buttons have `aria-label` attributes. The profile dropdown is keyboard-accessible (Enter/Space to open, Escape to close, arrow keys to navigate).
15. **Design tokens match Figma exactly:** per the Figma Design Summary section above — all colors, spacing, typography, border-radius, and shadow values.
16. **No regression** in existing functionality — sidebar navigation, task list, task details, timeline, auth flows, and timer widget all continue to work correctly.

## Dev Notes

### Current State
- The app currently uses a sidebar (`src/app/shared/components/sidebar/`) for primary navigation (implemented in Story 4.11).
- There is an old `NavigationComponent` (`src/app/shared/components/navigation/`) with a `mat-toolbar` that renders "TaskFlow" branding, user name, and a logout button. This is **no longer imported** in `app.ts` (replaced by sidebar in 4.11), but the component files still exist.
- The current `app.html` layout is: `sidebar | main-content`. This story adds the shellbar above that, making it: `shellbar` (top) → `sidebar | main-content` (below).
- User info and logout are currently in the sidebar bottom section — the shellbar will become the **primary location** for user profile & logout, potentially simplifying the sidebar.

### Figma Context
- The Header (node `7:3`) appears across all authenticated Figma page designs as a persistent top bar.
- It complements the Sidebar (node `7:42`) implemented in Story 4.11, creating the full app shell: shellbar on top + sidebar on left + content in the remaining area.
- The Figma layout hierarchy on authenticated pages is: `Body → Header (7:3) → Container → [Aside (sidebar) | Main (content)]`.
